The new design shape is:

- Resume Fit Analysis is a feature-owned module with `health()` and `analyze({ jobContent, jobPostingAnalysis, masterEvidenceItems })`.
- Resume Creation receives one Fit Score result and no longer knows the runtime's candidate-then-score protocol.
- The Python fit runtime remains an adapter behind the Resume Fit Analysis module.
- Lower-level `fitRuntimeClient` and `resumeLlm` injection remains for compatibility and tests.

The reported Create Resume failure was then diagnosed and fixed in:

- `src/features/resumes/ml/analysis.py`
- `src/features/resumes/ml/test/test_analysis.py`

The root cause was that candidate matching validated the LLM's short Job Content `evidence` phrase in Node, but the Python runtime scored only the longer requirement text. Broad requirement text diluted keyword and TF-IDF coverage enough that obvious Master Resume support could still classify as `weak evidence` or `no evidence`.

The fix uses the validated requirement `evidence` phrase as the runtime matching text when present, while preserving the full requirement text for signals and display context.
