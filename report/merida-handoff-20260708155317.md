Changed Resume Fit runtime files:

- `src/features/resumes/ml/analysis.py`
- `src/features/resumes/ml/test/test_analysis.py`

The reproduced failure class:

- A Fit Requirement can have a short evidence phrase validated against Job Content, but the concrete supported skill can live in the requirement text.
- Before the fix, the Python runtime mostly matched candidates against the short evidence phrase.
- If the evidence phrase was generic, such as `modern web applications`, but the requirement text named a concrete supported skill such as `Build React workflow interfaces`, real Master Resume evidence could still score as `no evidence`.
- That failure shape matches the user's `0/16` required/responsibility support and `0/31` total support symptom.

The fix:

- Candidate matching now includes both the validated evidence phrase and the full Fit Requirement text.
- Keyword coverage now uses the better match from full requirement text or evidence phrase.
- Evidence strength now treats normalized skill overlap plus meaningful lexical overlap as adjacent evidence.
- A regression test now covers generic evidence phrase plus concrete requirement text support.
