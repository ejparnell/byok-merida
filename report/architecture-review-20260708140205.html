Problem

The Job Postings backend module is shallow: its interface now includes Resumes adapters and Fit runtime startup.

Solution

Deepen an app-level Local Operator module and keep Job Postings and Resumes behind feature route adapters.

Benefits

  • locality: routes concentrate
  • leverage: one HTTP seam
  • Job Postings shrinks
  • tests lose resume fakes
ADR note: no direct contradiction if `/analysis` and `/resumes` remain separate; the move deserves a small ADR because old structure puts the shell under Job Postings.
