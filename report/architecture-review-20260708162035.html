Strong local-substitutable

Before

sequenceDiagram
  participant Shell as Local Operator
  participant Route as Job Postings adapter
  participant Batch as Analysis Batch Run
  Shell->>Route: request, response, corsHeaders, config
  Route->>Route: write NDJSON headers

After

sequenceDiagram
  participant Shell as Local Operator
  participant Route as Job Postings adapter
  participant Batch as Analysis Batch Run
  Shell->>Route: stream emitter at route seam
  Route->>Batch: emit(event)
  Batch-->>Route: event
  Route-->>Shell: event
  Shell->>Shell: NDJSON headers + CORS + end

Problem

The route adapter sees raw response mechanics, so the Local Operator seam is shallow for streaming.

Solution

Let Local Operator own NDJSON response mechanics and let Job Posting Analysis emit Analysis Batch Run events.

Benefits

  • locality: HTTP logic in shell
  • leverage: streaming is reusable
  • route adapter shrinks
  • tests use fake streams

Problem

Local Operator tests and startup still know low-level Notion, LLM, and runtime adapters.

Solution

Make the shell depend on route adapters while feature factories own their own lower-level adapters and config checks.

Benefits

  • locality: shell tests stay shell-shaped
  • leverage: adapter seam earns more
  • config ownership is clearer
  • feature fakes move inward
